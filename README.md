# crpt-openapi

Неофициальная OpenAPI-спецификация и Bruno-коллекции для API ГИС МТ «Честный ЗНАК» (ЦРПТ).

## Отказ от ответственности

Данный проект **не является официальным продуктом** ООО «Центр развития перспективных технологий» или ООО «Оператор-ЦРПТ». Спецификации и коллекции подготовлены на основе общедоступной документации и не гарантируют полноту, точность или актуальность. Используйте на свой страх и риск. Для получения официальной информации обращайтесь к первоисточнику.

## Первоисточник

Официальная документация API ГИС МТ: [docs.crpt.ru/gismt/Раздел_для_разработчиков/](https://docs.crpt.ru/gismt/%D0%A0%D0%B0%D0%B7%D0%B4%D0%B5%D0%BB_%D0%B4%D0%BB%D1%8F_%D1%80%D0%B0%D0%B7%D1%80%D0%B0%D0%B1%D0%BE%D1%82%D1%87%D0%B8%D0%BA%D0%BE%D0%B2/)

## Возможности

- **OpenAPI 3.0** спецификации для True API и ЭДО Лайт в формате YAML
- **Bruno-коллекции** с готовыми шаблонами запросов, заголовками, телом и окружениями

## Структура проекта

```
openapi/
  true-api.yaml      — Спецификация True API
  edo-light.yaml     — Спецификация ЭДО Лайт
bruno/
  true-api/          — Bruno-коллекция True API
    environments/    — Окружения (песочница/продуктив × API v3/v4)
  edo-light/         — Bruno-коллекция ЭДО Лайт
    environments/    — Окружения (продуктив/интеграция)
```

## API

### True API

Основной API ГИС МТ для участников оборота товаров. Покрывает:

- **Аутентификация** — получение ключа сессии через УКЭП
- **Подача сведений** — создание документов, агрегация, упаковка, резервирование партий
- **КИ и товары** — проверка, история движения, информация по GTIN, складские остатки, согласия
- **Документы и чеки** — статусы, содержимое, списки, валидация УПД
- **Квитанции** — результаты обработки документов
- **Выгрузки данных** — создание задач, статусы, скачивание ZIP-архивов
- **Национальный каталог** — карточки товаров, бренды, атрибуты, фиды
- **Станция управления заказами (СУЗ)** — токен аутентификации
- **ЭДО Лайт** — получение ID абонента и ZIP-архивов документов

### ЭДО Лайт

API электронного документооборота ГИС МТ. Покрывает:

- **Исходящие документы** — загрузка, подписание, печать, скачивание УПД/УПДи/УКД/УКДи
- **Входящие документы** — загрузка титулов покупателя, печать, скачивание
- **Квитанции** — аннулирование, подписание, скачивание JSON/XML
- **Универсальные сообщения** — обмен через XML
- **МЧД** — машиночитаемые доверенности

## Как использовать

### OpenAPI спецификации

Откройте файлы из `openapi/` в любом совместимом инструменте:

- [Swagger Editor](https://editor.swagger.io/)
- [Redoc](https://redocly.com/redoc)
- Сгенерируйте клиент через [OpenAPI Generator](https://openapi-generator.tech/)

### Bruno коллекции

1. Установите [Bruno](https://www.usebruno.com/)
2. Откройте коллекцию: `bruno/true-api` или `bruno/edo-light`
3. Выберите окружение в выпадающем списке
4. Настройте токен аутентификации в `opencollection.yml`
